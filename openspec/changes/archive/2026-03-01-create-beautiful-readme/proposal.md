## Why

Le projet manque actuellement d'un README attrayant et informatif qui présente clairement ses fonctionnalités, son installation et son utilisation. Un beau README améliore l'expérience des développeurs, facilite l'adoption du projet et renforce la crédibilité du code.

## What Changes

- Création d'un nouveau README.md à la racine du projet avec un design moderne et visuellement attrayant
- Structure organisée avec sections claires : présentation, fonctionnalités, installation, utilisation, contribution
- Utilisation de badges, emojis et mise en forme Markdown pour améliorer l'apparence
- Ajout d'images ou captures d'écran si pertinent
